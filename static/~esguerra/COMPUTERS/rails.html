Rails Reference Sheet

First it's assumed that you've already created your database in mysql. Once you have your database you can use ruby on rails to query the database. The first step that you have to take is to create a rails application, here we call it predrna because it's intended to show the predominant non-canonical base-pairs in the whole nucleic acid database (ndb): 
-bash-3.1$ rails predrna
The next step is to configure the database access. You do this by modifying the file database.yml, where you basically just tell it which database to use and the authorization that it should have. 
-bash-3.1$ cd predrna
-bash-3.1$ emacs config/database.yml
After you've created your application, you can create a scaffold and then modify it to display your tables. 
-bash-3.1$ cd predrna
-bash-3.1$ ruby script/generate scaffold Pair Predom
This command is going to create a folder name Predom which contains all the code necesary to run the web application, in Ruby this is called the controller and the model. In this case we are using the database pairs, created previously in mysql.
