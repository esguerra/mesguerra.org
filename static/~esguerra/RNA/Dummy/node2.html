Program Output

With 3-DNA we can get parameters evaluated for the single stranded mode and the double stranded mode.
The data obtained from such are:

Single Stranded Mode:

  • auxiliary.par
    - Local Base Reference Frames
  • bp_helical.par
    - Local Helical Parameters
  • bp_step.par
    - Local Step Parameters
  • ref_frames.dat

  • file.outs
    - RMSD of bases
    - Overlap Area
    - local base step parameters
    - Chain and chi Torsion angles
    - Sugar conformational parameters
    - P-P & C1´-C1  virtual bond distances
    - Helix Radius (P, O4  & C1 )
    - Position (Px, Py, Pz) and local helical axis vector (Hx, Hy, Hz)

Double Stranded Mode:

  • auxiliary.par
    - Same as above but per base-pair
  • bp_helical.par
    - Same as above but per base-pair
  • bp_step.par
    - Same as above but per base-pair
  • cf_7methods.par
    - Parameters from other softwares
  • ref_frames.dat
  • bp_order.dat
    - The base-pair information before reordering. Contains the numbered info of which are the bases belonging to the helical regions.
  • col_chains.scr
    - Prepares chain colors.
  • col_helices.scr
    - Assigns colors to helices.
  • bestpairs.pdb
    - All the base-pairs that it finds given constraints.
  • hel_regions.pdb
    - The regions that are considered as helical So they must be close to A-DNA, B-DNA or Z-DNA in one way or another, I would think.
  • hstacking.pdb
    - It seems like it puts them in the same reference frame so that they can be stacked
  • stacking.pdb
    - Similar to hstacking. Don't know yet the difference
  • file.out
    - All base-pair plus base-step parameters and more
